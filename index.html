Course goals

By the end of this program learners will be able to:

  • Understand how Cython bridges Python and C/C++.
  • Profile Python code, identify hotspots, and accelerate them with Cython.
  • Build, package, and distribute compiled Python extensions.
  • Use advanced features: nogil, memoryviews, calling C/C++ libs.
  • Create teachable materials and demos to show performance impact.
Estimated total time

~11–12 weeks

Self-paced. Each module lists suggested duration and exercises.

Curriculum — Modules & details

Goals

Get comfortable with what Cython is and compile your first module.

Topics
  • What Cython is; Python vs Cython vs C/C++
  • Install & setup: `pip install cython`, build with `setup.py` and `pyximport`
  • First `.pyx` file, compile, import
  • Simple benchmarks with `timeit`
Exercises
  1. Write a simple function (fibonacci or sum loop) in Python and in Cython; benchmark both.
  2. Explore `cython -a` for a tiny function and explain what you see.
Mini project idea

Micro-benchmarks: build a small page showing time comparisons for simple algos (fib, factorial, sum of list).

Goals

Learn Cython syntax, static typing, and type conversions.

Topics
  • `cdef`, `cpdef`, `def` differences
  • Declaring C variables: `int`, `double`, `char *`
  • Memoryviews vs Python lists vs arrays
  • Conversions between Python and C types
Exercises
  1. Convert a Python loop-heavy function to Cython with `cdef` variables.
  2. Measure how much static typing affects perf on a numeric loop.
Mini project idea

Implement a small numeric library (vector ops) using memoryviews and test against NumPy loops.

Goals

Profile real code and make targeted Cython optimizations.

Topics
  • Profiling: `cProfile`, `line_profiler`, `%timeit`
  • Bounds checks, wraparound removal, `cdef` locals
  • Memoryviews, efficient loops
  • GIL: `with nogil`, `prange`, OpenMP basics
Exercises
  1. Profile a slow function and identify hotspots.
  2. Use `cdef` and memoryviews to reduce runtime; measure speedup.
  3. Experiment with `with nogil` on a pure C-type loop.
Mini project idea

Build a parallel Monte Carlo simulation using `nogil` + `prange` and demonstrate scaling.

Goals

Call native C libs and use C++ classes from Python via Cython.

Topics
  • `cdef extern from` usage
  • Defining and passing C structs
  • Wrapping C++ classes and exception handling
Exercises
  1. Wrap a tiny C function (like a custom random or math helper) and call from Python.
  2. Wrap a simple C++ class and instantiate it from Python.
Mini project idea

Integrate a small C library (e.g., a fast RNG or compression routine) and expose it as a Python package.

Goals

Apply Cython to real Python projects and package extensions.

Topics
  • Refactor CPU-heavy scripts to Cython
  • NumPy + Cython integration
  • Packaging and building wheels (`pyproject.toml`, `setuptools`)
Exercises
  1. Pick a real script (web scraping parsing loop, heavy transform) and Cythonize it.
  2. Package your Cython module and test install in a clean venv.
Mini project idea

Create a small pip-installable extension (e.g., an image filter or a custom aggregator) and publish a test release.

Goals

Turn your knowledge into teachable content — demos, slides, and live benchmarking tactics.

Topics
  • How to explain GIL, nogil, and static typing
  • Live demo recipe: Python → Cython → C
  • Designing hands-on lab exercises
Exercises
  1. Create a 10–15 minute live demo (slides + code) showing a 5–10x speedup on a real problem.
  2. Write 3 step-by-step lab tasks for students to follow locally.
Mini project idea

Deliver a short workshop (recorded or live) and collect feedback; iterate on lesson clarity and demo speed impact.

Goals

Understand internals, debugging, and staying current.

Topics
  • How `.c` files are generated
  • CPython API interactions
  • Debugging Cython: `cython -a`, debug symbols
  • Keeping up with Cython & Python changes
Exercises
  1. Use `cython -a` on a large function and explain hotspots.
  2. Set up debug symbols and reproduce/trace a crash.
Mini project idea

Contribute a doc improvement or small PR to Cython or write a blog post breaking down a tricky internals topic.

Capstone projects & teaching materials

Capstone A — High-performance Data Pipeline

Cythonize a CSV/Parquet processing pipeline, optimize with memoryviews, and package as an extension used in a small CLI tool.

Deliverables: repo, benchmarks, installable wheel.

Capstone B — NumPy Kernel & Workshop

Implement a custom NumPy kernel (e.g., image filter), benchmark vs vectorized NumPy, and present a 30-min workshop.

Deliverables: demo, slides, recording.
